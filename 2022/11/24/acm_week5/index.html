[蓝桥杯 2013 国 C] 危险系数

题目描述

地道的多个站点间有通道连接，形成了庞大的网络。但也有隐患，当敌人发现了某个站点后，其它站点间可能因此会失去联系。
我们来定义一个危险系数 $DF(x,y)$：
对于两个站点 $x$ 和 $y(x\neq y),$ 如果能找到一个站点 $z$，当 $z$ 被敌人破坏后，$x$ 和 $y$ 不连通，那么我们称 $z$ 为关于 $x,y$ 的关键点。相应的，对于任意一对站点 $x$ 和 $y$，危险系数 $DF(x,y)$ 就表示为这两点之间的关键点个数。
本题的任务是：已知网络结构，求两站点之间的危险系数。

输入格式

输入数据第一行包含 $2$ 个整数 $n(2 \le n \le 1000)$，$m(0 \le m \le 2000)$，分别代表站点数，通道数。
接下来 $m$ 行，每行两个整数 $u,v(1 \le u,v \le n,u\neq v)$ 代表一条通道。
最后 $1$ 行，两个数 $u,v$，代表询问两点之间的危险系数 $DF(u,v)$。

输出格式

一个整数，如果询问的两点不连通则输出 $-1$。

样例输出

分析

  • 题中的关键点即去掉这个点便无法从x->y或y->x,故我们只需遍历图找到x->y的必经点。由于
    x->y可能不止一条路线，固我们选择dfs进行搜索。

  • 对于必经点的判断：当我们用dfs找到x->y的方案数时，若经过某点的次数等于方案数，则该点为必经点

题目描述

给出 $N$ 个点，$M$ 条边的有向图，对于每个点 $v$，求 $A(v)$ 表示从点 $v$ 出发，能到达的编号最大的点。

输入格式

第 $1$ 行 $2$ 个整数 $N,M$，表示点数和边数。
接下来 $M$ 行，每行 $2$ 个整数 $U_i,V_i$，表示边 $(U_i,V_i)$。点用 $1,2,\dots,N$ 编号。

输出格式

一行 $N$ 个整数 $A(1),A(2),\dots,A(N)$。

样例输入

样例输出

提示

  • 对于 $60\%$ 的数据，$1 \leq N,M \leq 10^3$。
  • 对于 $100\%$ 的数据，$1 \leq N,M \leq 10^5$。

分析

此题主要问题是如何遍历图能不超时,若正向搜索，会出现类似1->2->9->3->4的路线，也就意味着最大编号点位置不确定，
我们需要先搜索一遍找到最大编号，再通过搜索起点到终点的路线并将途经能到达的最大编号设置为第一次搜索找到的，花费时间会大大增加。
为了快速确定最大编号位置，我们可以反向存图与搜索，也就是从编号大的点往小的编号点搜索，这样可以确保所经过点的所能到达的最大编号点为起点，意味着只需搜索一次即可。

封锁阳光大学

题目描述

曹是一只爱刷街的老曹，暑假期间，他每天都欢快地在阳光大学的校园里刷街。河蟹看到欢快的曹，感到不爽。河蟹决定封锁阳光大学，不让曹刷街。

阳光大学的校园是一张由 $n$ 个点构成的无向图，$n$ 个点之间由 $m$ 条道路连接。每只河蟹可以对一个点进行封锁，当某个点被封锁后，与这个点相连的道路就被封锁了，曹就无法在这些道路上刷街了。非常悲剧的一点是，河蟹是一种不和谐的生物，当两只河蟹封锁了相邻的两个点时，他们会发生冲突。

询问：最少需要多少只河蟹，可以封锁所有道路并且不发生冲突。

输入格式

第一行两个正整数，表示节点数和边数。
接下来 $m$ 行，每行两个整数 $u,v$，表示点 $u$ 到点 $v$ 之间有道路相连。

输出格式

仅一行如果河蟹无法封锁所有道路，则输出 Impossible，否则输出一个整数，表示最少需要多少只河蟹。

提示

【数据规模】
对于 $100\%$ 的数据，$1\le n \le 10^4$，$1\le m \le 10^5$，保证没有重边。

分析

此题考虑如何确定一种布局是否可行，关于道路的封锁可以理解为：一个点占有河蟹后，与该点相接的边都删去。
而河蟹理想状况下是间隔占点，我们不妨通过dfs遍历图，以0 1 0 1…..交替的方式去标记整个图(当前点与当前点可访问的点的标记值不同,当前点可访问点的标记是相同的)，同时记录每个点相接边数。
如图:

对标记的交替可以采用^操作，即flag[to]=flag[now]^1 可将1变0，0变1

当遍历完图时，有河蟹的点理想状况下要么是1要么是0，故我们可以分别统计这两种情况下删去的边的数量
若删去的边数量等于图边的数量则可行；若大于，则说明有重复删除的边，即有河蟹相邻；若小于，则说明有道路未封锁

值得注意的是，题目所给图并不一定是连通图，故若不是连通图，我们需要拆成很多小块。由画图实践可发现规律成功封锁的布局中1或0设置河蟹都可以成功封锁，但题目要求求最少河蟹数，也就是说对于每张图我们只需记录最少河蟹的情况即可，最终将所有图河蟹数以及删除边数相加判断即可

为了迅速访问每张小图点，我们可将访问到的点放到vector里，方便迅速遍历该小图的最少河蟹数及删除的边数，统计完后clear。
