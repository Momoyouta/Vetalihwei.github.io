DFS

  • DFS大致为：
    从起点开始，以上下左右其中一个方向为优先搜索方向，若该方向可以前进则以该点为起始点继续以该规则进行搜索(函数嵌套)，若该方向不可以则搜索下一个方向。当四个方向都不可行时回溯到上一个点。
    • 对于访问过的点都要标记已访问。
    • 回溯点的建立只需在搜索该该方向的点后使该点标记为未访问
      结合函数递归，即可实现回溯

BFS

  • BFS大致为:
    • 利用队列先进先出特性，储存要搜索的点坐标
    • 对于被搜索的点(x,y),对其各个方向上的下一个未访问过的点进行判断，若未超过边界，为访问过且无障碍则将其加入队列中待搜索,然后将点(x,y)踢出队列，进行队列中下一个点搜索，直到找到目标点为止,当队列为空队列时表明无法到达该点。
    • 由于每个可访问点都进行搜索，所有不需要回溯点。

迷宫

题目描述

给定一个 $N \times M$ 方格的迷宫，迷宫里有 $T$ 处障碍，障碍处不可通过。
在迷宫中移动有上下左右四种方式，每次只能移动一个方格。数据保证起点上没有障碍。
给定起点坐标和终点坐标，每个方格最多经过一次，问有多少种从起点坐标到终点坐标的方案。

输入格式

第一行为三个正整数 $N,M,T$，分别表示迷宫的长宽和障碍总数。

第二行为四个正整数 $SX,SY,FX,FY$，$SX,SY$ 代表起点坐标，$FX,FY$ 代表终点坐标。

接下来 $T$ 行，每行两个正整数，表示障碍点的坐标。

输出格式

输出从起点坐标到终点坐标的方案总数。

样例输入

样例输出

提示

对于 $100\%$ 的数据，$1 \le N,M \le 5$，$1 \le T \le 10$，$1 \le SX,FX \le n$，$1 \le SY,FY \le m$。

分析

此题为简单的搜索，题目要求方案数固采用dfs
当搜索到终点时方案数+1，回溯到上一个点向其他方向继续搜索方案。

马的遍历

题目描述

有一个 $n \times m$ 的棋盘，在某个点 $(x, y)$ 上有一个马，要求你计算出马到达棋盘上任意一个点最少要走几步。

输入格式

输入只有一行四个整数，分别为 $n, m, x, y$。

输出格式

一个 $n \times m$ 的矩阵，代表马到达某个点最少要走几步（不能到达则输出 $-1$）。

样例输入

样例输出

数据规模与约定

对于全部的测试点，保证 $1 \leq x \leq n \leq 400$，$1 \leq y \leq m \leq 400$。

分析

根据bfs的搜索方式可知，bfs第一次访问到终点的路径为最短路径。而dfs所找出来的第一条路径受优先搜索方向干扰，未必为最短路径，固此题用bfs进行搜索。

对于步数的记录，点(x,y)到下一个点(m,n)所花费步数为1,故从起点到点(m,n)的步数等于起点到(x,y)的步数+1，运用递推思想即可求出每个点的最短路径所花费步数，如：mp[next_x][next_y]=mp[x][y]+1;

因为对于走过的点都记录有到该点的最少步数，故在进行点的判断时可先判断是否走过该点，若走过直接输出该步数，若未走过则再进行bfs，这样可以缩短运行时间,即：(mp[fx][fy]!=-1)?mp[fx][fy]:bfs(fx,fy)(根据题意，走不到输出-1，故-1也可以作为未记录过的标识)
